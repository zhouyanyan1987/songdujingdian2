# 🚀 快速部署指南

## 5分钟完成部署

### 第1步：创建GitHub仓库
1. 登录 [GitHub.com](https://github.com)
2. 点击右上角 "+" → "New repository"
3. 仓库名称：`classical-reading-assistant`（或任意名称）
4. 选择 "Public"
5. 点击 "Create repository"

### 第2步：上传文件
1. 进入刚创建的仓库
2. 点击 "uploading an existing file" 或 "Add file" → "Upload files"
3. **重要：将本项目的 `index.html` 文件拖入上传区域**
4. 确认文件名为：`index.html`（必须是小写的index.html）
5. 填写提交信息：`Initial commit - 经典诵读助手`
6. 点击 "Commit changes"

### 第3步：启用GitHub Pages
1. 在仓库页面，点击 "Settings" 标签
2. 滚动到左侧 "Pages" 选项
3. 在 "Source" 部分，选择 "Deploy from a branch"
4. 在 "Branch" 部分，选择 "main"
5. 在 "Folder" 部分，选择 "/ (root)"
6. 点击 "Save"

### 第4步：访问应用
1. 等待2-3分钟部署完成
2. 访问：`https://您的用户名.github.io/仓库名/`
3. 开始使用经典诵读助手！

## ⚠️ 重要注意事项

### 文件名要求
- **必须**使用 `index.html`（小写）
- 不能使用 `Index.html` 或 `INDEX.html`
- GitHub Pages 只认根目录的 `index.html`

### 常见错误
1. **文件名错误**：使用了大写字母
2. **文件位置错误**：文件没有放在根目录
3. **部署时间**：首次部署需要2-3分钟
4. **浏览器缓存**：清除缓存后重试

### 验证部署成功
访问您的GitHub Pages URL后，应该看到：
- 页面标题："📚 经典诵读助手"
- 8个经典书籍卡片
- 可以点击书籍进行阅读

## 🔧 如果遇到问题

### 问题1：页面显示空白
**解决方案：**
1. 检查文件名是否为 `index.html`
2. 清除浏览器缓存 (Ctrl+F5 或 Cmd+Shift+R)
3. 确认GitHub Pages已启用

### 问题2：显示404错误
**解决方案：**
1. 确认仓库为Public状态
2. 等待3-5分钟让部署完成
3. 检查Pages设置是否正确

### 问题3：功能不工作
**解决方案：**
1. 尝试不同浏览器（Chrome、Firefox、Safari）
2. 禁用浏览器插件
3. 检查网络连接

## 📱 移动端使用
- 应用完全支持手机和平板
- 自动适配不同屏幕尺寸
- 触摸操作友好

## 💾 数据安全
- 所有阅读数据保存在用户浏览器中
- 不会上传到任何服务器
- 清除浏览器数据会导致进度丢失
