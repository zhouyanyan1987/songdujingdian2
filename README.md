## 🚀 GitHub Pages 部署指南

### 方法一：GitHub 网页界面（最简单）

#### 步骤1: 创建GitHub仓库
1. 访问 [github.com](https://github.com) 并登录
2. 点击右上角的 "New" 按钮创建新仓库
3. 填写仓库信息：
   - **Repository name**: `classical-reading-enhanced`
   - **Description**: `📚 经典诵读助手 - 8本经典完整版，支持智能分章和进度跟踪`
   - 选择 **Public** (免费用户)
   - ✅ 勾选 "Add a README file"
4. 点击 "Create repository"

#### 步骤2: 上传应用文件
1. 在新创建的仓库页面，点击 "uploading an existing file"
2. 将本包中的 `index.html` 文件拖拽到页面上
3. 在 "Commit message" 框中输入：
   ```
   📚 经典诵读助手 - 完整优化版
   
   ✨ 核心功能：
   - 8本经典完整原文（道德经81章、易经64卦等）
   - 智能分章系统（手动+自动分章）
   - 章节标题完整显示
   - 用户自定义上传功能
   - 完整进度跟踪系统
   - 优雅中国风界面设计
   ```
4. 点击 "Commit changes"

#### 步骤3: 启用GitHub Pages
1. 进入仓库的 "Settings" 标签
2. 在左侧菜单中找到 "Pages"
3. 在 "Source" 区域选择 "Deploy from a branch"
4. 选择 "main" 分支和 "/ (root)" 目录
5. 点击 "Save" 保存设置
6. 等待1-2分钟，GitHub Pages会显示您的网站链接

#### 步骤4: 访问您的应用
- GitHub Pages 会提供一个类似这样的链接：
  `https://您的用户名.github.io/classical-reading-enhanced/`
- 点击链接即可使用您的经典诵读助手！

### 方法二：使用Git命令行

#### 前提条件
- 已安装 Git
- GitHub账户

#### 步骤1: 创建本地仓库
```bash
# 创建项目目录
mkdir classical-reading-enhanced
cd classical-reading-enhanced

# 初始化Git仓库
git init

# 添加HTML文件
cp /path/to/index.html .

# 设置初始提交
git add .
git commit -m "📚 经典诵读助手 - 完整优化版

✨ 功能特色：
- 8本经典完整原文（道德经81章、易经64卦、黄帝内经162篇）
- 智能分章系统（支持手动-----和自动分章）
- 章节标题完整显示功能
- 用户自定义经典上传
- 完整的学习进度跟踪
- 传统中国风界面设计"
```

#### 步骤2: 连接并推送到GitHub
```bash
# 添加远程仓库（替换为您的仓库地址）
git remote add origin https://github.com/您的用户名/classical-reading-enhanced.git

# 设置主分支并推送
git branch -M main
git push -u origin main
```

#### 步骤3: 启用GitHub Pages
1. 进入GitHub仓库的 "Settings" 标签
2. 点击左侧 "Pages"
3. Source选择 "Deploy from a branch"
4. 选择 "main" 分支和 "/ (root)" 目录
5. 保存设置并等待部署完成

## 🛠️ 故障排除

### 常见问题

**Q: 网站无法访问？**
A: 
1. 确保GitHub Pages已正确启用
2. 检查仓库设置中的Pages配置
3. 等待1-2分钟让部署完成
4. 确认浏览器没有缓存问题

**Q: 页面显示异常或空白？**
A: 
1. 检查浏览器控制台是否有错误信息
2. 确认HTML文件已正确上传到根目录
3. 尝试清除浏览器缓存后重新访问
4. 检查网络连接是否正常

**Q: 阅读进度丢失？**
A: 
1. 数据保存在浏览器本地，清除浏览器数据会导致丢失
2. 确认浏览器允许网站使用本地存储功能
3. 尝试使用无痕模式测试

**Q: 无法添加新书籍？**
A: 
1. 确保文本内容不为空
2. 检查文本格式是否正确
3. 尝试刷新页面后重新操作

**Q: 智能分章不准确？**
A: 
1. 尝试手动添加分章符号（-----、====）
2. 检查文本中的标点符号是否完整
3. 手动调整分章设置
